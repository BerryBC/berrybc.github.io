# 一、"小型" - 人类观察项目的诞生

## 1.1 项目背景

因公司最近推 `生成式AI` ，但因这些不及传统的 `AI` ( 如 声音识别、视觉识别、预测算法 等 ) 有落地场景

而且更多的只是一种背靠一种 **人有我有，宁愿没用，但不能没有** 的意识去推广 ( 就是先有技术，再寻求场景，说白了，跟以前的 `大数据` 、`元宇宙`、`区块链` 等一样 )

所以我就比以往更有意无意的尝试更多的使用各种 to C ( 或者是非特定场景下的 `AI 工具` )

**某天**，当我尝试用我的样子变成一个女生时，给我的感觉是跟现在网络上的那种 `网红脸` 差异很大

我想着是否可以做一个虚拟的社交媒体账号来为公司增加顾客曝光 ( 增加顾客对我司的印象 ) ？

## 1.2 选择小红书

选择平台的时候，曾想过 `微博`、`小红书`、`抖音` 等 **内容** 、**用户体量** 都适合的平台

但最终选择了 `小红书`

主要考虑有以下几点：

1. 小红书的 `用户群体` 跟我司的 `顾客群体` 高度相似
2. 恰逢，我的小红书因为我儿子没穿衣服的照片给封了 ( 历史的 Post 都不允许看 )
3. AI 无法很好的生成人像一致性程度高的视频
4. 微博没有很好的互动性，更多是一种官方平台宣传的公告栏

## 1.3 垂类内容选择

因为 AI 生成的人像 `很难准确` 的给出 `商品图` 或者 `商品细节` 展示，所以无法做直接 `好物内容`、`探店`

所以只能围绕最简单的 `生活分享`

## 1.4 研究的数据以及样本的偏向性

因为数据只是基于已经 `关注我` 或 `跟我互动` 的人 ( 且小红书没有很好的细节数据露出 )

且 `小红书` 也有自己的兴趣推荐算法，所以我无法考察全部人对我生产内容的反应

只能考察对我生产内容有兴趣的人的反应


# 二、AI 的作用观察

## 2.1 AI 的能力以及适配的工作

### 2.1.1 此研究 AI 的描述

AI 有很多种类型，从游戏 NPC 互动、工业机械臂操作、视频识别、声频识别 等等，都算是 AI

此处我仅说明 `生成式 AI` ，即各种 文字生成、图像生成 模型

### 2.1.2 AI 的能力

关于这些 AI 的能力，我个人只能总结是

> 无法满足 **精确返回** 的需求，但对 **无准确性要求** 的需求是很友好的

其实现在网络上很多，一键生成网站、APP、游戏 等

均会看到成品很粗糙

或者是同质化十分厉害

即使简单的 生图、生文 ，因为 AI 不会生成 `没见过` 或 `不知道` 的内容

所以对于使用者来说，除非描述的十分准确，或者需求十分简单

不然是完全无法满足工作的

### 2.1.3 AI 不能承接的工作

例如 `财务、审计 等` 数字准确性要求很高的工作，AI 是完全没办法满足的

仅仅是一个辅助参考

例如 `程序员、工程师 等` 有明确输入输出的工作，倒是可以承担一部分

可是，最麻烦的就是，代码生成一瞬间，修 Bug 一整年

看样子像是更快的产出结果，可是质量是完全灾难性的

### 2.1.4 AI 能承接的工作

非准确性内容输出

如 **低级** 的音乐、文字工作者、图画工作者等

这些本身就没有 `对与错` 的分别的工作，AI 是很适合的

不是说 AI 能生成多好的内容

但 AI 可以 `迅速的`、`大量的` 生成，后续再人工进行筛选就可以得到较为有用的内容

## 2.2 AI 在本项目的应用

AI 对于本项目的内容，其实就是两个

1. 图片生成
2. 文案生成

### 2.2.1 图片生成

其实该项目的一个大点就是想探知人们对于 AI 生成图片的识别度

这里发现一个很奇怪的例子

> 人脑对于有 `连贯性` 的内容，即使单个内容看起来 `假` ，但不影响因为连贯而被识别为 `真`

很多照片一眼 AI ，但人们还是会去接受这种是 `真的` ，因为我前后发了较多 `真假参半` 的内容

而人会觉得既然是有部分是真的那肯定都是真的

更奇怪的是有一个人一开始说是 AI 生成的

但后面看多了后，居然还一直问我是怎么锻炼的

### 2.2.2 文案生成

因为 AI 的文案生成实在太僵硬

或者是太 `一眼 AI 了`

我也懒得去写提示词或者示例

所以这方面的使用我是停止了的


# 三、小红书推荐

本来想这个展开说说的

但发现小红书的运营公司好像很多

而且我也没有足够多的数据去佐证

所以我只说一下几个表征

## 3.1 LBS

小红书的 LBS 权重挺重的

基本粉丝很少有外市的

如果没有标地点的话，又少很多曝光量

不过额外说一下，标注 `地铁` 、 `车站` 等全国共有的地方时

曝光量是一下子上去，个人估计这大概是一个 Bug ？

## 3.2 小红书的推荐召回机制

应该在前期冷启动时的权重很重

一开始刷到的不点赞，基本笔记曝光就没啥了

## 3.3 明面过审，实际不过审

在小红书有针对笔记的审核

但有些标识 `审核通过` 的笔记

在自我推荐里面是没有该笔记的

且推荐流量也是几乎没有

## 3.4 蹭热度

即某个话题热度较高

希望加上 `Tag` 或 加上对应的 `关键词`

是有一定用处的

但实际上很难的

例如写 `寿司郎` ，有多少人会专门去看寿司郎的 post ？

例如写 `小洛熙` ，这种话题很快就封了。

但可以写 `马拉松`、`明星名字` 或者某些 `景区名字`，这些会蹭到某些 搜索 以及 推荐曝光


# 四、人的观察

基于我的 `内容垂类` ，我能看到的主要基本就是这三种人

1. 找对象的人
2. 找一夜情的人
3. 想看美女的人

但基于前面说的，我的样本是有偏差的，仅总结一下我有访谈的对象 ( 并不代表所有人，仅是个别社会现象 )

## 4.1 找对象的人

很多直接发自己的资料 ( 如年龄、身高、收入、住所等信息 )

咨询为什么想在网络上找对象

都说自己社交圈不大，现实生活中没办法找到对象

的确

### 4.1.1 工作环境较难结识异性

对于一些没有外出沟通，且工作时长较长的工种 ( 且很多是现在高薪的岗位 )

周围都是一些以工作为主的同性 ( 例如财务、程序员、幼师、教师 等 )

> 在工作环境 `较多同性`，且休息时间也不多，工作环境又固定。导致他们很难认识到异性

### 4.1.2 无个人爱好

因为这一辈子的年轻人，都是一些从小就接受教育被灌输某个目标 ( 读好书，找好工，赚大钱 )

而没时间、没兴趣去发展个人爱好

### 4.1.3 反而低收入群体较少压力

因为低收入群体在社会上流动性较大

容易在不同的工作环境中认识很多的人

且认识的很多都没有对未来有太大的 `生活期盼`

导致不会有很大的负单，只是希望有个伴大家一起 `支撑一下对方` ，或 `更好的享受当下`

## 4.2 找一夜情的人

这个真的很恐怖

虽然我能理解，不同平台也有很多人有各种目的

但是，很赤裸裸的直接发自己小弟弟尺寸的人

就真的很恶心

传闻在其他的平台还直接发自己 `小弟弟` 的图

就十分可怕

还有些是在明面上没有太露骨，但聊了一下之后，就开始不停的约出来，司马昭之心很容易看透啊

## 4.3 想看美女的人

这些人就较好，也不会私聊一些太可怕的内容

就是可能有些过分留言

( 但我觉得也算好 )

## 4.4 关于小红书平台的总结

其实还算挺好的了

起码对人的保护还是做得够

如果有些露骨的内容会封禁之类

( 可能一开始就是对内容监管较严 )

很难相信在外网平台会怎样

# 五、舆论的偏向以及数据的操纵

让我想起当初的一句话

> 现在，一只松鼠死在你家门前可能比非洲有人死亡更让你关心。

基于此次实验

明显感觉到 `舆论的偏向` 以及 `数据的偏向` 是十分容易的

## 5.1 为啥美国不想本地媒体被外国控制？

当我在 Facebook 上发一篇针对某党派公众人物的不良内容时 ( 且一并放铁证 )

这篇文章在公众推荐流中均不被推荐 ( 只能点进我的主页才能看到 )

且公众推荐流中，该党派公众人物的正面形象内容被不断推荐

那么作为公众，是否会看到我的振臂高呼的内容？

这就是所谓的舆论操纵。

在当代，我甚至不需要逮捕你，我只需要让你在社交媒体上的曝光量降为0

你说了话，等于没说话。

## 5.2 数据的偏向以及信息茧房

关于推荐算法

我想问，当你 `只知道` 一个人买过 `腊肠` 时

你会推荐他买什么？

会是推荐 `口香糖` 么？

会是推荐 `香菜` 么？

不，最优的方法是，继续推荐 `腊肠`

因为推荐其他的时候，很容易会有 `失败的可能`

但对于 `腊肠` ，你已经成功过一次，所以你很有可能会继续成功。

这也是很多运营以及很多迷信 Golden Case 的人，风险最小的选择

但对于顾客来说，这是一个好的 `推荐` 么？

这变相对顾客是一种很大的 `可能性缺失` ，失去对更多更好选择的曝光

更有甚者，顾客看到只有 `腊肠` 的出现，甚至不知道会有其他选择

继续购买 `腊肠`

从而继续加深该顾客对其他 `可能性` 的曝光丢失
